# CheckMate

Java 기반 온라인 보드게임 동아리 관리 프로그램입니다.  
보드게임 동아리 운영에 필요한 회원 관리, 보드게임 정보 조회, 대여 관리, 회비 및 예산 관리, 게시판, 공지사항, 설문, 알림 기능을 데스크톱 GUI 환경에서 사용할 수 있도록 구현했습니다.

## Troubleshooting

### 1. 보드게임 대여 중복 방지

#### Problem

한 명의 회원이 여러 보드게임을 동시에 대여하거나, 이미 대여 중인 보드게임이 다시 대여되는 문제가 발생할 수 있었습니다.

#### Cause

대여 등록 시 기존 대여 상태를 확인하지 않고 데이터를 추가하면, 회원 기준 또는 보드게임 기준의 중복 대여가 발생할 수 있었습니다.

#### Solution

대여 등록 전에 `game_rental` 테이블에서 회원 ID와 보드게임 ID를 기준으로 현재 `대여중` 상태인 데이터가 있는지 먼저 조회했습니다. 이미 대여 중인 회원이거나 게임인 경우 대여 등록을 중단하도록 처리했습니다.

#### Result

회원별 1건 대여 제한과 보드게임 중복 대여 방지를 통해 대여 관리의 안정성을 높였습니다.

### 2. 회원 등록과 입회비 데이터 연동

#### Problem

회원 등록과 입회비 입금 내역이 따로 관리되면, 신규 회원의 회비 납부 상태를 별도로 입력해야 하는 번거로움이 있었습니다.

#### Cause

회원 정보는 `member` 테이블에 저장되고, 입금 내역은 `deposit` 테이블에 저장되기 때문에 두 데이터가 함께 처리될 필요가 있었습니다.

#### Solution

회원 등록이 성공하면 입력된 입회비 정보를 바탕으로 `deposit` 테이블에 입금 내역을 함께 저장하도록 구성했습니다. 입회비 값이 없거나 숫자로 처리하기 어려운 경우 기본값을 적용하도록 처리했습니다.

#### Result

회원 등록과 입회비 기록이 함께 처리되어 동아리 회원 관리와 회비 관리의 일관성을 높였습니다.

### 3. 회비 및 예산 현황 계산

#### Problem

입금 내역과 지출 내역이 각각 다른 테이블에 저장되어 있어, 현재 동아리의 총 자산을 한눈에 확인하기 어려웠습니다.

#### Cause

총 입금액은 `deposit` 테이블, 총 지출액은 `expenditure` 테이블에서 각각 계산해야 했습니다.

#### Solution

입금 합계와 지출 합계를 각각 조회한 뒤, 총 입금액에서 총 지출액을 차감하여 현재 총 자산을 계산하도록 구현했습니다.

#### Result

동아리 운영자가 입금, 지출, 총 자산 현황을 쉽게 확인할 수 있도록 개선했습니다.

### 4. 공지사항 등록과 회원별 알림 생성

#### Problem

공지사항을 작성해도 사용자가 직접 공지 목록을 확인하지 않으면 새 공지를 인지하기 어려웠습니다.

#### Cause

공지사항 데이터와 사용자별 읽음 여부가 분리되어 관리되어야 했기 때문에, 공지 등록 후 회원별 알림 데이터를 따로 생성할 필요가 있었습니다.

#### Solution

공지사항 등록이 성공하면 생성된 공지 ID를 기준으로 대상 회원들에게 `user_notifications` 데이터를 일괄 생성하도록 구성했습니다.

#### Result

공지사항 등록과 동시에 회원별 알림을 생성하여 공지 전달 흐름을 개선했습니다.

### 5. 설문 중복 투표 방지

#### Problem

같은 회원이 동일한 설문에 여러 번 투표하면 설문 결과의 신뢰도가 낮아질 수 있었습니다.

#### Cause

투표 수만 증가시키는 방식으로 처리하면 회원별 투표 여부를 확인하기 어렵습니다.

#### Solution

투표 전 `tblpolluser` 테이블에서 회원 ID와 설문 ID를 기준으로 기존 투표 기록을 확인했습니다. 투표 후에는 회원 ID, 설문 ID, 선택 항목 번호를 저장하여 투표 이력을 남기도록 구성했습니다.

#### Result

회원별 중복 투표를 방지하고 설문 결과의 신뢰성을 높였습니다.
